ShopSimulator – Advanced Data Structures Project

Console-based shop simulator using trees, stacks, and recursion to manage categories, items, and purchase history.

Overview

ShopSimulator is a console application developed for the Advanced Data Structures course. The goal of the project is to design a small shop system that uses different data structures in meaningful ways, rather than just focusing on visuals.

The program lets the user earn money through a Rock-Paper-Scissors minigame and then spend that money in a store divided into categories like clothes, food, and games. Behind the scenes, the project showcases how trees, stacks, and recursion can be used to organize and navigate data in a clean, structured way.

My Role

  • Worked on core shop logic and user flow for navigating categories and items.
  • Helped integrate tree-based structures to represent the shop’s category hierarchy.
  • Implemented or contributed to features tied to purchase history and item selection.
  • Collaborated on testing, debugging, and refining the user experience in the console.

Data Structures and Architecture

Tree for Categories

  • Shop categories (such as clothes, food, games) are stored in a tree structure.
  • Each category node can contain subcategories or items, making navigation intuitive.
  • Recursive traversal is used to move through the tree and display available options.

Stack for Purchase History

  • Recent purchases are stored in a stack, following a Last-In, First-Out pattern.
  • The stack makes it easy to review or potentially extend features like undoing recent purchases.

Object-Oriented Design

  • Classes are used to represent core concepts such as items, categories, and money management.
  • Encapsulation and modular design keep logic separated and easier to maintain.

Gameplay and Features

  • Rock-Paper-Scissors minigame that allows the player to earn money before shopping.
  • Shop menu divided into clear categories so the player can choose what to buy.
  • Balance tracking to ensure the player can only purchase what they can afford.
  • Purchase history stored for later inspection or future feature expansion.

What I Learned

  • How to apply advanced data structures (trees, stacks) in a practical application.
  • How recursion can simplify navigation and processing of hierarchical data.
  • How to design console-based user flows that remain clear and user-friendly.
  • How to collaborate on a shared C++ codebase and keep architecture consistent.
