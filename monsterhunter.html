Monster Hunter – Object Oriented 2 Project

Windows Forms C# project focused on OOP architecture, combat logic, and map exploration.

Overview

This Monster Hunter–style game was developed as part of my Object Oriented 2 course. The project focuses on applying core OOP principles in a practical context using Windows Forms and C#. The player explores a grid-based map, encounters monsters, and uses items and attacks to survive.

The design emphasizes clear architecture, separation of concerns, and reusable code rather than complex graphics, making it a solid showcase of object-oriented design.

My Role

  • Designed the overall class structure and object model.
  • Implemented player, monster, and item classes using inheritance and interfaces.
  • Created the combat logic, including attacks, health, and damage resolution.
  • Connected the game logic to the Windows Forms UI (buttons, labels, map display).

Technical Details

  • Language: C#
  • Framework: .NET (Windows Forms)
  • Paradigm: Object-Oriented Programming

Key Systems

  • Class hierarchy for entities (player, monsters, items) using inheritance and polymorphism.
  • Interface-based behaviors to keep systems flexible and decoupled.
  • Turn-based combat system with health, damage, and win/lose conditions.
  • Map and movement logic to explore tiles and trigger encounters.

What I Learned

  • How to design and implement a maintainable OOP architecture in C#.
  • How to connect backend game logic to a Windows Forms UI cleanly.
  • Working with events, state management, and updating the UI based on game state.
